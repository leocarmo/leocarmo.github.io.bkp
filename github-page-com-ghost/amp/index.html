GitHub Page com Ghost

by Leonardo do Carmo

2019-09-07

Para esquentar os motores do blog, nada melhor do que publicar como ele foi feito (:

Será uma breve publicação, utilizarei diversas referencias para quem tiver interesse em se aprofundar no assunto, aqui serei breve e objetivo.

Parte 1: Ghost

Para iniciar, você vai precisar ter uma versão suportada do NodeJS instalada e o Yarn ou NPM como gerenciador de pacotes. Para verificar as instalações e versões, utilize:

node -v
npm -v
yarn -v

Segundo passo é instalar o Ghost-CLI com o gerenciador de sua preferencia

npm install ghost-cli@latest -g

Depois de instalado, verifique: 

ghost -v

Agora vamos criar um novo diretório para o projeto e instalar o Ghost

mkdir my-ghost && cd my-ghostghost 
install local
ghost start

Já será possível acessar seu site na seguinte url: https://leocarmo.github.io/ghost

Basicamente é isso para se iniciar. Para se aprofundar mais sobre o Ghost, recomendo a documentação oficial. Basicamente seguimos os passos de lá, é bem simples.

Parte 2: Github Pages

Agora é preciso criar um novo repositório em seu Github seguindo este padrão de nome: username.github.io

Com este repositório criado, vamos clona-lo para dentro daquela pasta que instalamos o Ghost com o nome de static:

# dentro da pasta my-ghost
git clone https://github.com/username/username.github.io static

Ainda dentro da pasta de instalação no Ghost, você verá um arquivo de configuração chamado config.development.json, altere o valor da chave url para https://username.github.io, alterando para o seu usuário criado.

Parte 3: Deploy

Com seu blog feito, é hora de publicar. Para isso vamos precisar de um gerar de site estático, e vamos utilizar o Buster.

Para instala-lo, precisamos do pip, um gerenciador de pacotes do Python. Para verificar se ele está instalado, utilize:

Instale o Buster via pip e verifique:

pip install buster
buster --version

Agora vamos configurar o Buster:

Para facilitar, criei um arquivo para agilizar o processo de deploy, por mais que seja bem simples. Para fazer isso, crie um arquivo com touch deploy.sh  e jogue este conteúdo dentro

Agora basta executa-lo com ./deploy.sh e visulizar seu site em: https://username.github.io

Para visualizar se o deploy foi realizado com sucesso, acesse a aba de deployments no seu projeto: https://github.com/username/username.github.io/deployments.

Todos os arquivos dentro da pasta static serão enviados para seu repositório e processados.

Dica 1: aguarde alguns minutos, na primeira publicação pode ser que o Github não tenha processado seu site ainda.

Dica 2: realize o backup do diretório onde instalamos o Ghost, o banco de dados do seu blog e todos os arquivos necessário para gerar o site estático ficam ali. Sem eles, tudo será perdido, então não esqueça de armazenar de forma segura.
